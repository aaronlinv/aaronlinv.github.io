JavaEE开发模式

模式在开发过程中总结出的“套路”，总结出的一套约定俗成的设计模式

  • model1模式
    技术组成：JSP+JavaBean
    弊端：
    业务多时，导致JSP页面容易混乱
  • model2模式
    技术组成：JSP+Servlet+JavaBean
    优点：
    开发中使用各个技术擅长的方面
    Servlet擅长Java的业务代码
    JSP擅长页面的展示
    MVC：
    Model:JavaBean 封装数据
    View:JSP 页面展示
    Controller:Servlet 获取数据，对数据进行封装，传递数据、指定页面显示

  • 三层架构
    Web层 与客户端交互
    Service层 复杂业务逻辑
    DAO层 与数据库进行交互
